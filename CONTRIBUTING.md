# Contributors

Thank you for your interest in contributing to **agent-greenroom**!

This project follows **Spec-Driven Development**. All changes must be driven by the formal specification.

## How to Contribute

### 1. Understand the Spec
- Read the current specification: [`spec/SPEC.md`](spec/SPEC.md)
- All features, bug fixes, and improvements must align with this spec.

### 2. Update the Specification First
Before submitting any code changes, you **must**:

1. **Modify** the existing spec, **or**
2. **Add** a new section to the spec

This ensures that:
- The project has a single source of truth
- All implementation stays aligned with documented requirements
- Contributors and maintainers share the same understanding

### 3. Submit a Pull Request
A valid PR **must** include:

- Changes to `spec/SPEC.md` (new requirements, clarifications, or updated behavior)
- Implementation that matches the updated spec
- Clear explanation of *why* the spec was changed

**PRs that do not update or extend the specification will be considered invalid and closed.**

---

## Development Workflow

1. Read `spec/SPEC.md`
2. Open an issue or discuss proposed spec changes
3. Update the specification first
4. Implement the feature to match the new spec
5. Open a Pull Request

We practice **Spec-Driven Development** — the spec leads, the code follows.

---

## Questions?

Feel free to open an issue if you're unsure how to update the spec or what should be included.

Happy contributing!
